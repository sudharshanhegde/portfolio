Brief summary about mysql.

MySQL More Like Your SQL 😂

MySQL is a database language used for syntax definition, data definition, data manipulation, etc.

Why is MySQL Important?

It is like a superpower for managing data. We store everything in a structured format, as its name suggests—Structured Query Language (SQL).

Creating, Using, and Manipulating Data in the Database

As always, we start with the first step, which is creating the database. It’s like a digital filing cabinet where we organize related data.

Step 1: Creating the Database

To create a database, use the following SQL command: 

CREATE DATABASE my_online_shop;

This will create the database.

Step 2: Using the Database

To use the database we just created, we use: 

USE my_online_shop;

This helps us access the database.

Step 3: Creating Tables

Tables have rows and columns, like a spreadsheet in our database.

Example: Creating a "users" Table

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100),
    signup_date DATE
);

Explanation:

  • id INT AUTO_INCREMENT PRIMARY KEY → The primary key ensures uniqueness and is automatically incremented.
  • name VARCHAR(50) → Can store up to 50 characters.
  • email VARCHAR(100) → Stores emails.
  • signup_date DATE → Stores the date of signup.

Foreign Key: A primary key from another table, used to establish relationships between tables.

Step 4: Inserting Data into the Table

To insert a record into the "users" table: 

INSERT INTO users (name, email, signup_date)
VALUES ('bete', 'bete@gmail.com', '2024-10-12');

Manipulating Data in SQL

Retrieving Data

Get all users:

SELECT * FROM users;

Get only specific columns (name and email):

SELECT name, email FROM users;

Get a user by signup date:

SELECT name FROM users WHERE signup_date = '2024-10-01';

Order users alphabetically:

SELECT * FROM users ORDER BY name ASC;

Updating Data

If there is a typo in the name, update it using: 

UPDATE users SET name = 'king' WHERE name = 'beta';

Deleting Data

Delete a specific user (ID = 100):

DELETE FROM users WHERE id = 100;

Advanced SQL Queries

Joining Tables

We have users and orders tables, and we need to join them using user_id: 

SELECT users.name, orders.total
FROM users
INNER JOIN orders ON users.id = orders.user_id;

Grouping Data

To count users by country: 

SELECT country, COUNT(*) AS user_count
FROM users
GROUP BY country;

Altering Tables

Add a new column (phone number):

ALTER TABLE users ADD COLUMN phone VARCHAR(15);

Modify the phone column size:

ALTER TABLE users MODIFY COLUMN phone VARCHAR(12);

Drop a column:

ALTER TABLE users DROP COLUMN phone;

Dropping Databases and Tables

Drop an entire database:

DROP DATABASE my_online_shop;

Drop only a table (users):

DROP TABLE users;

Deleting All Data from a Table

Delete all rows but allow undo:

DELETE FROM users;

Delete all rows (irreversible action):

TRUNCATE TABLE users;

Creating a View

Views act as virtual tables that simplify complex queries and restrict access to sensitive data. 

CREATE VIEW active_customers AS 
SELECT name, email 
FROM users 
WHERE status = "active";

Additional Information

  • Unique Keys: Ensures unique values but allows NULL values.
  • Composite Key: A primary key made by combining two or more columns.
  • Indexes: Speed up searches in specific columns.

Example: Creating an Index

CREATE INDEX idx_email ON users(email);
