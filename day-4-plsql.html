PL/SQL

Procedural language extensions to sql

Procedural language extensions to sql in short pl/sql is portable and high performance language designed to enhance the sql's procedural capabilities.

Lets say we are building a transactional system or a web application which will have complex database operation then PL/SQL will provide the advanced and robust tools.

**What is PL/SQL? ** Its a procedural oriented programming language developed by oracle to bridge the gap between the SQL's declarative nature and procedural programming needs.

It helps developers by enabling:- 1.execute complex logic within the database. 2.It reduces the network traffic by bundling the SQL statement into the blocks. 3.it leverages the features like error handling,encapsulation and object oriented programming.

Features and Advantages of the PL/SQL

  1. it helps in tight sql integration in such a way that it executes the queries and DML operations directly.

  2. it has built in exception handling for robust applications

  3. It organizes the code into procedures functions and packages so there is modularity.

  4. We can create types,inheritance and polymorphism so we can say that there is object oriented programming support.

  5. Its used in building the server pages and web applications.

PL/SQL combines querying and data transformation in one language which increase the productivity and also it has granular access which is controlled via roles and privileges.

Syntax and Structure of the PL/SQL:-

It is an block structure language . Each block consists of three sections which are declare begin and end.

Declare : these is used to declare variables,cursors and types.

begin: these is mandatory and it marks the beginning of the executable statements.

exception : these is used for error handling.

end: these marks the end of the code block.

lets take an example

DECLARE

 MESSAGE VARCHAR2(20) := 'WELCOME TO MY BLOG';

BEGIN 

 DBMS_OUTPUT.PUT_LINE(MESSAGE);

END;

THESE WILL RETURN THE OUTPUT AS WELCOME TO MY BLOG.

Data types in PL/SQL are :-

numeric data types examples are NUMBER, PLS_INTEGER,BINARY_FLOAT.

character data types examples are CHAR,VARCHAR2,CLOB ETC.

Boolean data type example is BOOLEAN (TRUE,FALSE,NULL)

date/time data type example is DATE,TIMESTAMP.

Exception handling in PL/SQL
	
the code will be as follows.


DECLARE
     dividend NUMBER := 10;
		 
		 divisor NUMBER := 0;

		     result := dividend / divisor;

Working with strings and arrays in the PL/SQL we have some sample functions like TRIM which removes the empty spaces from the string provided and SUBSTR which gives us our desired substring when we specify the starting and end point of the string. we can use the UPPER function to make all the characters in upper case and similarly all in lower case by using LOWER function.

FOR i IN 1 . . names.COUNT LOOP

Cursor in PL/SQL:-

It is used to fetch data from queries into variables.

Syntax of the cursors is as follows:

DECLARE

CURSOR c_customers IS SELECT name FROM customers;
	
	c_name customers.name %TYPE;

BEGIN

	  OPEN c_customers;
		
		LOOP
		
		    FETCH c_customers INTO c_name;
				
				EXIT WHEN c_customers % NOTFOUND;
				
				DBMS_OUTPUT.put_line('customer' || c_name);
				
		END LOOP;
	  
		CLOSE c_customers;

END;

Stored Procedures:-

It is used to create reusable code blocks:

syntax of the stored procedures will be as follows:

CREATE OR REPLACE PROCEDURE raise_salary(emp_id NUMBER,amount NUMBER) IS

BEGIN

 UPDATE employees SET salary = salary + amount WHERE id = emp_id;
	 
	 COMMIT;

END;
