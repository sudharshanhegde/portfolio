Data Link Layer

The data link layer is responsible for ensuring reliable communication between directly connected devices. It handles:

  • Framing
  • Error detection
  • Flow control
  • Media access management

Point-to-Point Protocol (PPP)

A protocol used to establish and maintain direct connections between two nodes, primarily for serial communications like dial-up and DSL connections.

Features:

  • Encapsulation:

    • Encapsulates network layer data for serial link transmission
    • Supports multiple network layer protocols (IPv4, IPv6)

  • Authentication:

    • Supports PAP (Password Authentication Protocol)
    • Supports CHAP (Challenge Handshake Authentication Protocol)

  • Error Detection:

    • Uses CRC (Cyclic Redundancy Check)
    • Ensures data integrity

  • Multilink Support:

    • Combines multiple physical links into one logical connection
    • Increases throughput (Multilink PPP)

Advantages:

  • Compatible with various network protocols (IP, IPX, AppleTalk)
  • Enhanced security through built-in authentication
  • Optional compression improves efficiency
  • Simple configuration for point-to-point links

Applications:

  • Establishing home computer-to-ISP connections
  • DSL connections between users and ISPs

High-Level Data Link Control (HDLC)

A bit-oriented protocol for both point-to-point and multipoint communication that ensures error-free data transmission over synchronous links.

HDLC Frame Structure:

  1. Flag Field: Marks frame start/end (ensures synchronization)
  2. Address Field: Identifies recipient/station
  3. Control Field: Manages flow/error control
  4. Payload: Contains transmitted data
  5. CRC: Provides error detection

Communication Modes:

  1. Normal Response Mode (NRM):

    • Secondary nodes transmit only when requested
    • Used in master-slave configurations

  2. Asynchronous Balanced Mode (ABM):

    • Both nodes can initiate communication
    • Enables peer-to-peer communication

Advantages:

  • Error detection via CRC
  • Effective flow control prevents congestion
  • Optimized for synchronous links (low overhead)

Applications:

  • Reliable data transfer over leased lines
  • Industrial control/automation networks

Spanning Tree Protocol (STP)

A network protocol that prevents loops in Ethernet networks by creating loop-free topologies while maintaining redundancy.

How STP Works:

  1. Root Bridge Election:

    • Switches exchange BPDUs (Bridge Protocol Data Units)
    • Switch with lowest Bridge ID becomes root bridge

  2. Path Cost Calculation:

    • Each switch calculates shortest path to root bridge
    • Based on assigned link costs

  3. Port Roles:

    • Root Port: Provides shortest path to root bridge
    • Designated Port: Forwards traffic to network segments
    • Blocked Port: Prevents loops (standby state)

  4. Topology Updates:

    • Switches continuously exchange BPDUs
    • Adapts dynamically to network changes

Features:

  • Enables backup links for failover
  • Prevents broadcast storms and packet loops

Advantages:

  • Maintains loop-free topology
  • Reduces network disruptions
  • Automatically adapts to topology changes

Applications:

  • Enterprise networks with redundant paths
  • Mission-critical high-availability configurations
