Network Address Translation

Overview on NAT, DHCP and DNS

Network Address Translation (NAT)

NAT is a method used by routers and firewalls to modify IP addresses in data packets as they traverse between private and public networks. It allows multiple devices on a private network to share a single public IP address, ensuring efficient usage of the IPv4 address space.

Types of NAT

  • Static NAT: Maps one private IP address to a single public IP address. Commonly used for internal servers hosting web applications or other services that require consistent external access.

  • Dynamic NAT: Maps private IP addresses to a pool of available public IP addresses. Allows multiple devices to access the internet without requiring a unique public IP for each device.

  • Port Address Translation (PAT) (also known as NAT Overload): Maps multiple private IPs to a single public IP address using unique port numbers. Common in home and small office networks where multiple devices share one public IP.

Advantages of NAT

  • Conserves public IP addresses
  • Improves security by hiding internal IP addresses
  • Enables internet access for private networks

Limitations of NAT

  • Latency: Address translation can introduce communication delays
  • Compatibility Issues: May interfere with protocols that require direct end-to-end connectivity

Dynamic Host Configuration Protocol (DHCP)

DHCP automates the process of assigning IP addresses and other configuration details to devices in a network. It eliminates the need for manual configuration and ensures efficient IP address usage.

How DHCP Works

  1. Discovery: The client broadcasts a request for an IP address when it joins the network.

  2. Offer: The DHCP server responds with an available IP address and other network settings.

  3. Request: The client requests to lease the offered IP address.

  4. Acknowledgement: The server confirms the lease and assigns the IP address to the client.

Key Components

  • Server
  • Client
  • Lease Time

Advantages of DHCP

  • Automates IP address management
  • Reduces configuration errors
  • Scalable for large and dynamic networks

Limitations of DHCP

  • Highly dependent on the DHCP server
  • Potential for misconfiguration

Domain Name System (DNS)

DNS translates user-friendly domain names into numerical IP addresses required for computer communication.

How DNS Works

  1. Query: The user enters a domain name in the browser.

  2. Resolution: A recursive resolver queries the root, TLD, and authoritative DNS servers to find the domain’s IP address.

  3. Response: The DNS server returns the IP address of the requested domain.

  4. Connection: The browser uses the resolved IP address to connect and retrieve content from the web server.

Common DNS Record Types

  • A Record: Maps domain names to IPv4 addresses
  • AAAA Record: Maps domain names to IPv6 addresses
  • CNAME Record: Maps one domain name to another (aliases)
  • MX Record: Specifies mail servers for email delivery

Advantages of DNS

  • User-Friendly: Allows users to access websites using domain names instead of IP addresses
  • Scalability: Hierarchical structure supports the size and growth of the internet
  • Redundancy: Distributes queries across multiple servers to reduce downtime

Limitations of DNS

  • Vulnerabilities: Susceptible to attacks like DNS spoofing and cache poisoning
  • Server Downtime: Can disrupt access if DNS servers are unavailable
