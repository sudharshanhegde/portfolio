day 3 transport layer protocols.

Various elements of transport layer protocols.

Building blocks of Transport layer protocols.

Transport layer is a crucial component of network communication it acts as a bridge between the applications we daily use and underlying network infrastructure. It ensures data is delivered efficiently reliably and securely between the devices.

  1. Connection Management:-

TCP :- it uses the three way handshake (syn,syn-ack,ack) these will set up the reliable connection it will ensure both reciever and sender are ready before the data transfer begins.

UDP:- it is connectionless means there is no handshake it just sends the data its like your freind who sends reels on and on even if you watch or not watch.These is very fast and very unreliable.

2.Reliability:-

TCP :- It implements using acknowledgements and retransmissions so if the data is lost or not send we will resend it.

UDP :- These has no reliability guarantees and packets can be lost duplicated and arrive out of order basically anything can happen and its not responsible for anything.

3.Congestion control:

TCP :- It will use algorithms like slow start and congestion avoidance so that we can adjust transmission rates based on the networks conditions.

UDP:- It has no congestion control. It can contribute in network congestion if it is used carelessly.

4.Flow control:-

TCP:- it uses sliding window mechanism and reciever says how much data can be sent befor waiting for another acknowledgement.

UDP:- well there is no such thing as flow control here.

5.Multiplexing and Demultiplexing:

These directs data to the correct application using the port numbers.

ports are 16 bit identifiers(0-65535) in these (HTTP = 80 and HTTPS = 443) Both TCP an UDP will use the ports to ensure the data reaches the right process.

6.Error checking:

TCP/UDP :- both uses the checksums to verify the data integrity. TCP goes further by repairing errors via retransmissions but the UDP just discards packets without any recovery.

7.Segmentation and Reassembly:

Breaking of large data chunks into smaller segments and reconstructions of them in the destination.

TCP: segments are numbered for ordered delivery.

UDP:- it treats each segment as independent datagrams.

Use cases

TCP is used in web browsing ,file transfers etc.

UDP is used in streaming ,gaming ,VoIP etc.

TCP is like a first bench topper who keeps everything organized and does everything in a proper way while UDP is like that last bench student who pulls the all nighter for the exam and somehow takes as good marks as the toppers. Use cases for both of them is different they are like humans only one prefers order while the other prefer chaos.
