API:-Application programming Interface.

API:- short for Application programming Interface is something that lets app talk to each other without exposing their code.

HOW DOES THE API WORK?

API runs on simple cycle which is ask, process and answer.Well these is simple lets say if we are trying to post something on lets say instagram and then phone sends a request to instagram API telling can i post these picture of mine. Then the instagram API checks if i am a verified user and if i have the permission to post then it will be like yeah sure go ahead. And then the post goes live as simple as these it is gonna work.

Building blocks of APIS

API has few key parts namely

  1. Endpoints
  2. HTTP methods
  3. Requests and Responses.

What are Endpoints?

Well lets think of it as a postbox at every village where all the postcards are gonna be sent. Its like a unique pincode something you use to send API does also have a unique url which helps us to request the data from the field that we want.

What are HTTP methods?

There are four brothers who come along together and have some particular things to be done.

GET:-These fetches the data.Like loading those instagram posts which you obviously scroll for hours.

Post:- These is how we post our data. Like uploading your lousy profile pictures on instagram (no offence)

put/patch:- These is how we update our existing data. Like all those happy to share that... posts on linkedin which makes you scared that you are not going anywhere.

Delete :- These will make things vanish.Thanos snapped to kill half of the universe.Well these does the full.Data ok not universe.

Requests and Responses

API takes our question or what we call a request and returns a response usually in gods second language JSON the format which both humans and machines can read.

Sizes of the APIs

Well like most things APIs come in different models:

  1. Public API :- These are some of those free or paid tools anyone can use like google maps api which lets you find location from find us page.
  2. Private API :- these are those behind the scene tools which are used internally in the companies like Amazons warehouse robots working together in coordinated way.
  3. Partner API :- These is something that is shared between partner companies something like netflix or amazon prime sharing API with smart TV brands.
  4. Composite APIs :- These are the multitaskers.These are something that bundles the multiple requests in to one like your favourite delivery apps having those meal deals where you can get all that you want.

REST vs SOAP

These is like fight between BIG SHOW and MARK HENRY..

REST API :-It is a flexible options.It uses simple HTTP requests and return easy to read JSON. Most of the APPs use these.

SOAP API:-These send XML data with strict protocols.These are used in banking and healthcare where security is very important.

HOW does the API Rule the world
They are in Automation, they collobarate and integration brings the more powerful yet to be born and they are doors to the innovation where engineers are working seamlessly to reduce the fetch time..
