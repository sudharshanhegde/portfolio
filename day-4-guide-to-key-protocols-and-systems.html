Day 4 guide to key protocols and systems

Guide to Key protocols and Systems.

Fundamental components of Internet.

1.Domain Name System(DNS)(port 53 UDP/TCP)

These translates the human readable domain names into machine readable IP addresses.

Key features:-

It supports hierarchial structure with root Toplevel domain and authoritative servers.

It supports the record types like A(IPV4),AAAA(IPV6),MX(mail servers) and CNAME(aliases)

without DNS we have to memorize IP addresses to access websites.

2.Hypertext Transfer Protocol(HTTP :- port no 80 ,HTTPS :-port no 443):-

it governs the communication between web browsers and servers to fetch resources(HTML,images etc)

Key features of the HTTP:-

Stateless :- No memory of previous interactions (sessions are managed using the cookies)

it uses the methods like get(retrieve data),post(send data) and put(update data)

HTTPS adds encryption via SSL/TLS for security.

Status codes:- 200 ok :- success. 404 not found:- resource missing. 500 internal server error :- it will be a server side issue.

3.Telnet(port number 23)

It is a legacy protocol for remote terminal access to devices over the network.

Key features:-

unencrypted:- these will transmits data in plaintext (insecure). it is largely replaced by SSH(secure shell) which encrypts traffic.

These is rarely used today except for debugging or legacy systems.

4.File transfer protocol(FTP port number 21)

These is used to transfer files between the client and server.

features of the FTP are:-

it uses two channels : control (commands) and data(file transfers).

active vs passive mode: it determines how the data connections are established.

security flaw:- the transmitted credentials and files are in plain text so modern alternatives have arrived such as SFTP and FTPS to encrypt the data.

5.World wide web:-

System of interlinked hypertext documents accessed via the internet.

components are:- HTTP :- it is protocol for fetching the resources. HTML :- language for structuring the contents for the web. URLs(uniform resource locator):- these are the addresses to locate the resources. WWW is the service on the top of the internet.

6.Simple mail transfer protocol(SMTP)

it sends and routes the emails between the servers.

1.it operates on the port 25.

2.works with POP3(download emails) and IMAP(sync emails across devices) 3.modern extensions like STARTTLS are used to encrypt email traffic.

These protocols are the reason why our web works so smoothly.
