Ethernet and wifi

802.3 Ethernet

It is a popular wired networking technology used in LANs. It provides a framework for data communication and ensures seamless interoperability across devices from various manufacturers.

Features:

  • Operates at the data link and physical layers of the OSI model
  • Delivers high-speed and reliable communication
  • Utilizes unique MAC addresses to identify devices within a LAN

How It Works:

  1. Data is encapsulated in Ethernet frames which include:
    • Source and destination addresses
    • Payload
    • CRC for error detection
  2. Uses Carrier Sense Multiple Access with Collision Detection (CSMA/CD) to:
    • Manage data transmission in shared networks
    • Prevent collisions
  3. Data is transmitted using electrical signals over various media types:
    • Twisted pair cables
    • Coaxial cables
    • Fiber optics

Advantages:

  • Highly reliable
  • Provides consistent, low-latency communication for critical applications
  • Accommodates a wide range of speeds to meet modern network needs
  • Cost-effective

Applications:

  • Enterprise LANs
  • Data centers
  • Home networks

802.11 (Wireless Networking Standards)

Commonly known as Wi-Fi, it eliminates the need for physical cables and enables devices to connect using radio waves.

Key Features:

  • Functions at the data link and physical layers
  • Manages MAC addresses and wireless signal transmission
  • Uses electromagnetic frequencies:
    • Primarily 2.4 GHz and 5 GHz bands
  • Allows devices to connect and move within network coverage areas without losing connectivity

How It Works:

  • Access points act as intermediaries, bridging wireless and wired networks
  • Uses radio waves (2.4 GHz, 5GHz, and 6 GHz) for transmission
  • Ensures secure communication using encryption protocols:
    • WPA2
    • WPA3

Ethernet Switching

A networking technology that connects multiple devices within a LAN. It operates at the second layer of the OSI model. Ethernet switches facilitate efficient communication by directing data frames to their intended destinations using MAC addresses.

Unlike hubs that broadcast data to all devices, switches forward data only to the appropriate device, optimizing network performance and reducing congestion.

How It Works:

  • MAC Address Table:

    • Switches maintain a table that maps MAC addresses to specific ports
    • The table is dynamically populated by observing source MAC addresses of incoming frames
    • When a frame arrives, the switch looks up the destination MAC address and forwards the frame to the corresponding port

  • Frame Forwarding Methods:

    • Unicasting: Sent to a specific device based on MAC address
    • Broadcasting: Sent to all devices (e.g., ARP requests)
    • Multicasting: Sent to a specific group of devices

  • Learning Mechanism:

    • Switches learn MAC addresses of connected devices by analyzing source MAC addresses of incoming frames
    • Eliminates the need for manual configuration

  • Loop Prevention:

    • Uses Spanning Tree Protocol (STP) to:
      • Prevent broadcast storms
      • Identify and disable redundant network paths

Types of Switches:

  • Unmanaged Switches:
    • Basic plug-and-play operation
    • No configuration options
  • Managed Switches:
    • Offer advanced features:
      • VLAN configuration
      • Quality of Service (QoS)
      • Network monitoring
    • Suitable for enterprise environments requiring precise traffic control

Virtual Local Area Networks (VLANs)

A method of grouping devices within a LAN to allow communication as if they were on the same physical network, even when connected to different switches. VLANs segment networks into smaller, isolated broadcast domains to improve efficiency, security, and manageability.

How They Work:

  • Segmentation:

    • Devices within a VLAN share the same broadcast domain
    • Confines broadcast traffic to the VLAN
    • Reduces unnecessary traffic and enhances performance

  • Tagging:

    • Uses IEEE 802.1Q standard to identify VLAN-specific traffic
    • Tags are added to Ethernet frames by switches
    • Tags are removed before frames reach end devices

  • Inter-VLAN Routing:

    • Requires a Layer 3 device (typically a router)
    • Routes traffic between VLANs based on IP addresses

Types of VLANs:

  • Default VLAN:
    • Includes all devices by default
  • Data VLAN:
    • Carries standard user data traffic
  • Voice VLAN:
    • Dedicated to voice traffic
    • Ensures high-quality communication by prioritizing voice packets
  • Management VLAN:
    • Reserved for managing network devices
    • Provides secure access for administrative tasks
