Fourth Normal Form :- tackling multivalued dependencies

March 26, 2025

day 3 fourth normal form

dbms4NFnormalization

Fourth normal form handles the issue of multivalued functional dependencies.

multivalued dependency occurs when determinant is associated with multiple independent set of values.

Lets say there is a table where there is employee id skill and language for every employee the skill is also dependent on employee id and language is also dependent on employee id.These skills and language is independent attributes which are stored in the same table these will create redundant data and update anamolies.

A table is in 4NF if:- if it is boyce-codd normal form(BCNF) if it has no nontrivial multivalued dependencies.

So what i will do is i will seperate it to two different table in one table there will be employee and skills and in another table there will be employee and languages.

benifits of it is there will be no redundancy it will be easy to update and avoid the anomalies.

4NF is critical when it contains two or more independent multivalued attributes and when you need to eliminate combinatorial redundancy.
