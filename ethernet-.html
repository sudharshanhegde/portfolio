Ethernet Frames, ARP, and Network Access Control

Ethernet Frames, ARP, and Network Access Control

Ethernet Frame – The Building Block of Ethernet Networks

An Ethernet frame is the basic unit of communication in Ethernet networks. It encapsulates data and moves it between devices in a well-structured format to ensure smooth and efficient delivery.

Why Ethernet Frame is Important:

  • It organizes and transmits data in a structured format
  • It carries essential metadata for communication

Components of an Ethernet Frame:

  1. Preamble (7 bytes) A pattern of alternating 0s and 1s used to synchronize the sender and receiver clocks, signaling the start of a frame.

  2. Start Frame Delimiter (1 byte) Follows the preamble and indicates where the actual frame data begins.

  3. Destination MAC Address (6 bytes) Physical address of the device that is supposed to receive the frame.

  4. Source MAC Address (6 bytes) Physical address of the sender’s device.

  5. EtherType (2 bytes) Identifies the protocol being used, like IPv4 or IPv6. In some variants, it can represent payload length.

  6. Data (46–1500 bytes) The actual payload or information being transmitted.

  7. Frame Check Sequence (4 bytes) Contains a CRC (Cyclic Redundancy Check) for detecting errors. The receiver recalculates it to ensure data integrity.

Ethernet Frame Variants

1. Ethernet II Frame

This is the standard used in modern networks. It includes the EtherType field to indicate higher-layer protocols like IPv4 or IPv6. It's the default for TCP/IP communication.

2. IEEE 802.3 Frame

Instead of EtherType, it uses a length field. It’s less common in today’s Ethernet setups.

3. VLAN Tagging

In VLAN-enabled networks, a 4-byte VLAN tag is inserted between the source MAC address and EtherType field. This identifies the VLAN and enables logical separation of networks.

Key Functions:

  • Data encapsulation
  • Error detection
  • Addressing
  • Flow control

Address Resolution Protocol (ARP)

ARP works at the Data Link Layer (Layer 2) and helps map a logical IP address to a physical MAC address. It's essential for enabling communication within a local subnet.

How ARP Works:

  1. Request When a device wants to communicate, it sends a broadcast asking "Who has this IP?".

  2. Response The device with the requested IP replies with its MAC address in a unicast reply.

  3. Cache Devices store resolved mappings in an ARP cache to avoid repeating the process.

Types of ARP:

  • Gratuitous ARP – Sent without being requested, usually for updating other devices after IP/MAC changes.
  • Proxy ARP – A router answers ARP requests on behalf of another device, helping with communication across subnets.
  • Reverse ARP (RARP) – Converts MAC addresses back to IP addresses.

ARP Security Concerns

ARP Spoofing

A malicious actor sends fake ARP messages, making it appear as if their MAC address matches that of a legitimate device (like a server or router). This can lead to data theft or manipulation.

Cache Poisoning

Incorrect entries are added to a device’s ARP cache, causing data to be misrouted, potentially to an attacker.

Network Access Control (NAC)

NAC is used to secure a network by managing who and what can connect to it. It ensures only authenticated, authorized, and compliant devices are granted access.

How NAC Works:

  1. Authentication Verifies the identity of users or devices.

  2. Authorization Determines access levels based on identity and compliance.

  3. Compliance Check Verifies if the device meets security standards (e.g., antivirus status, system updates).

  4. Ongoing Monitoring NAC continues to monitor device behavior and compliance throughout the connection session.

Types of NAC:

  • Pre-admission NAC – Evaluates devices before granting access to the network.
  • Post-admission NAC – Monitors device activity after access is granted and can restrict access if policy violations are detected.

Key Components:

  • Policy Server Central system defining access rules and evaluating requests.

  • Enforcement Points Network devices like routers, switches, and firewalls enforce access based on decisions from the policy server.

  • Authentication Methods NAC supports protocols like 802.1X, RADIUS, and LDAP for validating users and devices.

Applications of NAC:

  1. Corporate Networks – To manage employee and guest access.
  2. Educational Institutions – For securely connecting students and staff devices.
  3. Healthcare Systems – To ensure patient data is accessed only by compliant devices.
  4. IoT Security – NAC protects smart devices that connect to the network.
