Docker

What is Docker?

Docker is a tool that packages an application and all its dependencies—including code, libraries, configurations, and OS-level components—into a portable unit called a container.

Containers can run consistently across any environment: on your laptop, a cloud server, or another machine. This ensures identical behavior regardless of the underlying system.

Why Use Docker?

  1. Consistency: Containers run identically everywhere, simplifying debugging.
  2. Dependency Isolation: Resolves "works on my machine" issues by bundling exact library versions, eliminating conflicts.
  3. Environment Isolation: Runs multiple apps with conflicting setups (e.g., Python 2 vs 3) side-by-side without interference.

How Docker Works

Docker uses a client-server architecture:

  • Docker Engine (daemon): Runs on the host OS to manage containers, images, networks, and volumes.
  • Docker Client: CLI tool to interact with the Engine.

Key Components:

  1. Image:
    A read-only blueprint of your app—a snapshot of all dependencies. Built from a Dockerfile, it’s immutable and portable.
  2. Container:
    A live, isolated instance of an image. Can be started, stopped, or destroyed without affecting the host OS.
  3. Dockerfile:
    A script defining steps to build an image (e.g., installing dependencies, copying code).
  4. Docker Hub:
    A cloud registry (like GitHub) for storing and versioning Docker images.

Containers vs. Virtual Machines (VMs)

Feature Virtual Machine (VM) Docker Container
OS Full OS per VM Shares host OS kernel
Resource Usage Heavy (GBs of RAM/Disk) Lightweight (MBs)
Boot Time Minutes Seconds
Isolation Hardware-level Process-level
Portability Complex Simple (via images)

Example: Dockerfile Explained

FROM node:18          # Base image: Node.js v18
WORKDIR /app          # Sets working directory to /app
COPY package*.json ./ # Copies dependency files
RUN npm install       # Installs dependencies
COPY . .              # Copies app code
EXPOSE 3000           # Opens port 3000
CMD ["node", "app.js"] # Runs the app

Build & Run:

docker build -t my-node-app .   # Builds image from Dockerfile
docker run -p 3000:3000 my-node-app  # Runs container, maps host port 3000 → container port 3000

Essential Docker Commands

Command Description
docker ps Lists running containers
docker ps -a Lists all containers (running/stopped)
docker images Lists all images
docker run -it ubuntu Starts interactive Ubuntu container
docker exec -it bash Opens shell in a running container
docker stop Stops a container
docker rm Deletes a stopped container
docker rmi Deletes an image

Docker Volumes

Persist data beyond container lifecycle: 

docker volume create mydata              # Creates a volume
docker run -v mydata:/app/data myapp    # Mounts 'mydata' to /app/data

Volumes are managed by Docker and survive container deletion.

Docker Compose

Manages multi-container apps (e.g., app + database) via a declarative docker-compose.yml file: 

docker-compose up   # Starts all services
docker-compose down # Stops and removes everything

.dockerignore

Exclude files/directories (like node_modules or secrets) from being copied into images. Works like .gitignore.

Key Benefits Recap

  • Reproducibility: Eliminates "it works on my machine" problems.
  • Isolation: Prevents dependency conflicts between apps.
  • Efficiency: Containers share the host OS kernel, reducing overhead.
  • Portability: "Build once, run anywhere" consistency.
