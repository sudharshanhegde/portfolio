IP Addressing

IP addressing is used to uniquely identify devices on the network and enable efficient routing and delivery of data packets. It works on the network layer and enables seamless communication across local and global networks.

Why is it needed?

  1. It provides unique identification for each device in the network and ensures there is no two devices sharing the same address.
  2. It facilitates the routing of the data packets across the interconnected networks and enables global communication.
  3. It defines what is LAN and what is WAN or the internet.

IPv4

It is the fourth version of the Internet Protocol. It uses 32-bit numerical address to uniquely identify devices on the network. It is written in dotted decimal notation â€” each address is divided into four octets separated by periods with values ranging from 0 to 255.

IPv4 Header

It consists of several fields that enable the data transmission, routing, and control. Its minimum size is 20 bytes and can extend up to 60 bytes with optional fields included.

Components of IPv4 Header:

  • Version: Specifies the version being used (4 for IPv4). Ensures that packet is interpreted correctly by the receiving devices.
  • Header Length: Indicates the length of header in 32-bit words. Helps identify where the actual payload data begins.
  • Type of Service: Provides how the packet should be handled in terms of priority and quality of service, minimizing delay or maximizing throughput.
  • Total Length: Specifies the total size of the packet, including header and payload data, in bytes. Maximum value is 65,535 bytes.
  • Identification: Unique identifier assigned to each packet to aid in reassembling fragmented packets back into the original message.
  • Flags: A set of three bits used to control packet fragmentation â€” whether the packet can be fragmented or whether more fragments are expected.
  • Fragment Offset: Indicates the position of the current fragment in relation to original data, ensuring fragments are correctly reassembled.
  • Time to Live (TTL): Specifies the maximum number of hops a packet can traverse before being discarded. Prevents indefinite looping in the network.
  • Protocol: Identifies the protocol used in the data portion of the packet (e.g., TCP or UDP).
  • Header Checksum: A value used to detect errors in the header. If checksum doesn't match at the router, the packet is discarded.
  • Source Address: Specifies the device from which the packet originated.
  • Destination Address: Specifies the device to which the packet is being sent.
  • Options: Provides additional functionality like routing preferences or timestamps. Not commonly used due to performance impact.

Address Classes in IPv4

  • Class A: Designed for large networks with millions of devices. Range: 1.0.0.0 to 126.255.255.255 Used by ISPs and multinational companies.

  • Class B: Suitable for medium-sized networks. Range: 128.0.0.0 to 191.255.255.255 Used by universities and large organizations.

  • Class C: Used for smaller networks in small businesses. Range: 192.0.0.0 to 223.255.255.255

  • Class D: Reserved for multicast transmissions. Range: 224.0.0.0 to 239.255.255.255 Used for streaming and conferencing applications.

  • Class E: Reserved for experimental purposes. Range: 240.0.0.0 to 255.255.255.255

Limitations of IPv4:

  • IPv4 provides 4.3 billion unique addresses, which is insufficient for modern global requirements.
  • No built-in encryption or authentication, making data transmissions vulnerable.
  • Rapid internet growth has led to near exhaustion of IPv4 addresses.

It is used in LANs, WANs, and early internet infrastructure to assign addresses to devices.

IPv6

It uses 128-bit addresses to overcome limitations of IPv4 and provides an unlimited address pool to support modern internet demands. Addresses are written in colon-hexadecimal notation. The address consists of eight 16-bit segments separated by colons and can include shorthand notation for consecutive zero segments.

IPv6 is designed to be simpler and more efficient than IPv4. Header size is 40 bytes, which reduces processing overhead and enhances routing performance.

Components in IPv6 Header:

  • Version
  • Traffic Class
  • Flow Label
  • Payload Length
  • Next Header
  • Hop Limit
  • Source Address
  • Destination Address

Subnetting

Subnetting is the process of dividing a larger network into smaller, more manageable sub-networks (subnets). It is useful for organizing networks, reducing congestion, and optimizing IP address allocation.

Why It Is Used:

  • Reduces network congestion by limiting broadcast traffic within smaller subnets.
  • Enhances network security by restricting unauthorized access and isolating sensitive data.
  • Optimizes IP address usage by avoiding waste in networks that don't need large address pools.

How Does It Work:

  • Subnet Mask: A value that separates the network and host part of the IP address. Example: 255.255.255.0 indicates first 24 bits are for network and remaining for hosts.

  • CIDR Notation: Specifies number of bits used for the network. Example: 192.168.1.0/24 means 24 bits for network and 8 bits for host addresses.

Supernetting

Supernetting combines multiple smaller networks into a larger one. It is used for route aggregation to simplify routing and reduce the size of routing tables.

Benefits:

  • Reduces routing table size by aggregating routes and reducing entries in the router table, improving performance.
  • Optimizes routing efficiency by combining contiguous networks into a single entity.
  • Simplifies network management and allows administrators to manage fewer networks while maintaining connectivity.

How It Works:

  • Combines contiguous subnets into a larger network.
  • Uses a shorter subnet mask to cover a broader range of addresses.

Classful vs Classless Addressing

Classful Addressing:

  • Divides IP addresses into fixed classes (A, B, C, D, E) with predefined subnet masks.
  • Leads to inefficient IP allocation in larger networks.
  • Easy to understand and suitable for small networks with predictable needs.
  • Disadvantage: Wastes IP addresses due to fixed subnet sizes, lacks flexibility for modern networks.

Classless Addressing:

  • Eliminates rigid boundaries of classful addressing.
  • Allows Variable Length Subnet Masking (VLSM) for more efficient IP allocation.
  • Requires more complex calculation and planning compared to classful addressing.
