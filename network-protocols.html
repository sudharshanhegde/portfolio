Network Protocols and OSI Layers

Network Protocols and OSI Layers

Network Protocols

Definition: Network protocols are standardized rules that govern how data is transmitted and received over a network. They ensure reliable, organized, and secure data exchange between heterogeneous devices.

Types of Protocols

  1. Communication Protocols – HTTP, SMTP, FTP
  2. Transport Protocols – TCP, UDP
  3. Routing Protocols – OSPF, BGP
  4. Security Protocols – SSL, IPsec

Key Protocols and Their Roles

TCP (Transmission Control Protocol)

  • Connection-oriented, reliable
  • Uses 3-way handshake
  • Ensures ordered delivery, error checking, retransmission
  • Has flow control and congestion control
  • Used in: web browsing, file transfer, emails

UDP (User Datagram Protocol)

  • Connectionless, no guarantee of delivery or order
  • Fast and low-latency
  • Used in: live video/audio streaming, online gaming, VoIP

HTTP (Hypertext Transfer Protocol)

  • Used for data transfer between client and server (web browsing)
  • Uses port 80
  • Stateless protocol: no memory of previous requests
  • Requires cookies/tokens for session tracking
  • REST methods: GET, POST, PUT, DELETE

HTTPS (Secure HTTP)

  • HTTP with SSL/TLS encryption
  • Used for secure web transactions

FTP (File Transfer Protocol)

  • Transfers files between client and server
  • Ports: 20 (data), 21 (control)
  • Active mode: server connects to client
  • Passive mode: client connects to server
  • Requires username/password
  • Used in data backup and file sharing

SFTP (SSH File Transfer Protocol)

  • FTP with encryption via SSH
  • Secure file transfer

SMTP (Simple Mail Transfer Protocol)

  • Used for sending emails between mail servers
  • Ports: 25 (standard), 465/587 (secure)
  • Only handles sending, not receiving (uses IMAP/POP3 for that)

SNMP (Simple Network Management Protocol)

  • Used for monitoring and managing network devices

  • Ports: 161 (queries), 162 (notifications)

  • Components:

    • Managed Devices
    • SNMP Agents
    • SNMP Manager

Network Layer (OSI Layer 3)

Functions

  1. Assigns logical IP addresses

  2. Routing – optimal path selection (Dijkstra’s, Bellman-Ford)

  3. Packet forwarding via routers

  4. Fragmentation/Reassembly of large packets

  5. Key Protocols:

    • IPv4 (32-bit), IPv6 (128-bit)
    • ICMP – error reporting and diagnostics (e.g., ping)
    • ARP – resolves IP to MAC addresses
    • BGP – routing between autonomous systems (internet-scale)

Transport Layer (OSI Layer 4)

Functions

  1. Segmentation and reassembly of data

  2. Error detection and recovery

  3. Flow control to avoid congestion

  4. Port addressing for service identification

  5. Protocols:

    • TCP – reliable, connection-oriented
    • UDP – fast, connectionless
    • SCTP – hybrid of TCP and UDP features

Session Layer (OSI Layer 5)

Functions

  1. Establishes, maintains, and terminates sessions
  2. Synchronizes data exchange with checkpoints
  3. Supports full-duplex/half-duplex communication

Protocols

  • NetBIOS – session services for communication
  • PPTP – used in VPNs for tunneling
  • SOCKS – proxy protocol for secure and anonymous sessions
